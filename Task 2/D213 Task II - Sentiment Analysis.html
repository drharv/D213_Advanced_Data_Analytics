C-2. Discuss the number of layers, the type of layers, and total number of parameters.¶

We are employing 5 layers in our model.

  • First is the Embedding layer with 64K parameters, a result of the 5400 words and embedding length of 12
  • Second is a Flatten layer to accompany the Embedding layer
  • Third is a Dropout layer in an attempt to prevent overfitting
  • Fourth is a Dense layer with a 13.6K parameters, a results of the 852 input values and 16 nodes
  • Fifth is the final later with 17 parameters - a result of the input values

C-3. Justify the choice of hyperparameters, including the following elements:¶

  • activation functions
  • number of nodes per layer
  • loss function
  • optimizer
  • stopping criteria
  • evaluation metric

1. Activation Functions

We are employing two different activation functions. The first is a Rectified Linear Unit. This function returns zero for values below zero, and returns the value itself above zero. ReLU is appropriate for most scenarios (Chen, 2021).

The second activation function is Sigmoid. We are using this in the final layer because we are moving towards a binary classification. The outputs for Sigmoid range between 0 and 1.

2. Nodes per Layer

Our first layer is dictated by the number of categoies (words) in our model. For the fourth Dense layer we are using 16 nodes. It is an arbetrary number between the first and last layer node counts, and appeared to work well with the model.

The final layer is using 1 node, as we try to get the data ready for a binary classification.

3. Loss Function

Our loss function is Binary Crossentropy. This is the appropriate loss function when there are two possible output values.

4. Optimizer

We are using Adam for optimization. The Adam optimizer combines the two primary advantages of other optimizers - Adaptive Gradient Algorithm, and Root Mean Squared Propogation (Brownlee, 2021a).

5. Stopping Criteria

We are employing early stopping during the fitting of our model. We will base the stopping on the min loss value. If we haven't seen consistently equal or lower values for loss after four epochs, we will revert back to the last epoch that showed progress.

6. Evaluation Metric

Our primiary evaluations are Accuracy and Loss. Loss equates to the error that we seek to minimize with a loss function, and Accuracy is the ratio of correct predictions to the entire set of targets.

Stopping criteria can prevent a model from overfitting. In addition, stopping criteria can prevent the model from using unnecessary compute cycles. After the model reaches the optimum state, it will stop processing and ignore further epochs.
